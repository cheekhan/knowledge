# 综合知识库软件 — 需求设计文档

版本：v2.0 ｜ 平台：Windows / macOS ｜ 技术栈：Electron + React + TypeScript

> v2.0 变更：删除 Wiki 双链/反链/内容级 PDF 引用语法；PDF 与笔记的关联改为**文件级绑定（一对多）**，由 sidecar 中的 `boundNotes` 列表维护。

---

## 1. 产品概述

一款本地优先（Local-First）的桌面知识库软件，核心能力：

1. Markdown 笔记，以本地文件树形式存储
2. PDF 阅读与手动打标签（含扫描版 PDF）
3. PDF 与笔记**文件级绑定**（一对多），左右分屏"边读边记"

设计原则：

- **纯文件是唯一事实来源**：笔记就是磁盘上的 `.md` 文件，用户可用任何编辑器打开，不被软件绑架
- **元数据 sidecar 化**：标签、批注、绑定关系不写进 PDF 原文件，存于同名 sidecar JSON
- **索引可重建**：SQLite 仅作搜索索引，损坏后可从纯文件全量重建
- **绑定关系简洁**：PDF 与笔记的关联不解析笔记内容，只看 sidecar 中的 `boundNotes` 列表
- **单实例运行**：同一时刻只允许一个进程操作同一 Vault，避免并发写损坏 sidecar（见架构 4.8）

### FR-6 笔记 ↔ PDF 绑定与分屏

**核心模型**：PDF 与笔记是**一对多绑定关系**——一份 PDF 可绑定多篇笔记；一篇笔记至多绑定一份 PDF。绑定关系不解析笔记内容，只由 sidecar 中的 `boundNotes: ["相对路径/note.md"]` 列表维护。

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| FR-6.1 | **分屏视图**：左 PDF 阅读器、右 Markdown 编辑器，宽度可拖（react-resizable-panels），支持互换左右、关闭一侧 | P0 | 布局状态记忆（比例、左右位置） |
| FR-6.2 | **创建绑定笔记**：阅读 PDF 时点击工具栏"创建笔记"→ 输入笔记名（默认 `《PDF名》笔记 N`，N 自动递增）→ 在指定目录创建 `.md`（默认 `notes/` 下与 PDF 同名的子文件夹）→ 路径写入 sidecar `boundNotes` → 自动进入分屏，右侧打开该笔记 | P0 | 新笔记自动带 `# 《PDF名》笔记` 标题；重复点击创建多篇，N 递增 |
| FR-6.3 | **打开绑定笔记**：文件树 PDF 节点下展示绑定的笔记列表；或 PDF 侧栏"关联笔记"面板中选择；点击 → 分屏模式，左侧保持 PDF，右侧打开该笔记 | P0 | — |
| FR-6.4 | **解除绑定**：右键绑定笔记 →"解除绑定"→ 从 sidecar `boundNotes` 移除，笔记文件保留在磁盘 | P1 | 解除后文件树中该笔记回归普通笔记位置 |
| FR-6.5 | **绑定路径自动维护**：重命名/移动笔记或 PDF 时，sidecar `boundNotes` 中的路径自动更新；PDF 被删除时，其绑定笔记变为普通笔记 | P1 | 移动后绑定关系不丢失，面板数据实时正确 |
| FR-6.6 | **绑定面板**：PDF 阅读器侧栏"关联笔记"面板，列出该 PDF 全部绑定笔记，支持新建 / 打开 / 解除绑定 / 在文件树中定位 | P1 | — |
| FR-6.7 | **创建绑定笔记后 PDF 保持渲染**：点击"创建笔记"完成绑定后，若 PDF 路径未变，PDF 不应重新加载（仅恢复已销毁的 engine 实例） | P0 | 分屏切换不丢 PDF 渲染状态 |

| 类别 | 指标 |
|------|------|

## 4. 数据模型

### 4.1 磁盘结构（示例库）

```
MyVault/
├── notes/
│   ├── 读书笔记.md
│   └── 项目/
│       └── 设计草稿.md
├── pdfs/
│   ├── 深入理解计算机系统.pdf
│   └── 深入理解计算机系统.pdf.meta.json   ← sidecar
├── assets/                                ← 笔记图片
│   └── img-20260720.png
└── .kb/                                   ← 软件内部目录
    ├── index.db                           ← SQLite FTS5 索引（可删可重建）
    └── app.json                           ← 库级配置（主题、最近打开等）
```

### 4.2 sidecar 元数据（`xxx.pdf.meta.json`）

```json
{
  "version": 2,
  "file": "深入理解计算机系统.pdf",
  "tags": ["计算机", "经典教材"],
  "lastPage": 42,
  "boundNotes": [
    "notes/深入理解计算机系统/《深入理解计算机系统》笔记 1.md",
    "notes/深入理解计算机系统/第三章习题整理.md"
  ],
  "updatedAt": "2026-07-20T08:00:00Z",
  "annotations": [
    {
      "id": "a1b2c3",
      "page": 12,
      "type": "rect",
      "rect": { "x": 0.12, "y": 0.30, "w": 0.60, "h": 0.08 },
      "color": "#FFD54F",
      "comment": "虚拟内存的核心思想",
      "createdAt": "2026-07-20T08:00:00Z"
    }
  ]
}
```

> - `rect` 为归一化坐标（相对页面宽高），保证任何缩放下对齐；扫描版与文本型 PDF 使用同一结构。
> - `boundNotes` 存**库内相对路径**，是绑定关系的唯一事实来源；打开 PDF 时读取此列表即可展示关联笔记，无需解析任何笔记内容。
> - 一篇笔记出现在某 PDF 的 `boundNotes` 中即视为绑定；从列表移除即解绑，笔记文件本身不受影响。

### 4.3 SQLite 索引表（`.kb/index.db`，可重建）

```sql
CREATE TABLE notes (
  path TEXT PRIMARY KEY,      -- 相对路径
  title TEXT,
  mtime INTEGER
);
CREATE VIRTUAL TABLE notes_fts USING fts5(title, content, content='notes', content_rowid='rowid');

CREATE TABLE pdfs (
  path TEXT PRIMARY KEY,
  name TEXT,                  -- 文件名（展示用）
  tags TEXT,                  -- JSON 数组冗余，便于检索
  mtime INTEGER
);

-- 绑定关系冗余表：加速"查某笔记绑定的 PDF / 某 PDF 的绑定笔记"
-- 事实来源仍是 sidecar，此表随索引重建
CREATE TABLE bindings (
  pdf_path TEXT,
  note_path TEXT,
  PRIMARY KEY (pdf_path, note_path)
);
```

> v2.0 变更：删除 `links` 表（原用于双链与 PDF 引用解析），绑定关系直接由 sidecar `boundNotes` 提供，`bindings` 表只是查询加速冗余。PDF 文件名不再要求库内唯一（绑定按路径而非名称）。

## 5. 关键用户流程

**流程 A：边读边记（核心路径）**
1. 文件树双击 PDF → 全屏阅读模式打开
2. 阅读中点击工具栏"创建笔记" → 确认笔记名（默认 `《PDF名》笔记 1`）→ 自动创建 md 文件并写入 sidecar `boundNotes` → 右侧滑出编辑器，进入分屏
3. 直接在右侧记笔记（内容无需任何特殊语法），自动保存
4. 日后再打开该 PDF → 侧栏"关联笔记"面板列出全部绑定笔记 → 点击任一篇即分屏对照阅读

**流程 B：扫描版 PDF 打标签**
1. 导入扫描 PDF → 阅读器正常渲染（无文本层不提示错误）
2. 侧栏添加文件级标签 `扫描件` `合同`
3. 用矩形工具框选关键段落，填写备注 → 存入 sidecar
4. 标签面板点击 `合同` → 列出所有该标签 PDF

**流程 C：重建索引**
设置页"重建索引"→ 清空 `index.db` → 全量扫描 md/pdf/sidecar → 重建 FTS 与 `bindings` 冗余表。
